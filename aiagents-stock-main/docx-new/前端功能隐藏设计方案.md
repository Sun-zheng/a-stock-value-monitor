# 前端功能隐藏设计方案

## 1. 需求分析

### 1.1 核心需求
- 实现前端功能入口的隐藏功能
- 主要隐藏以下功能模块的前台入口：
  - "根据不同策略筛选优质股票"（选股板块）
  - "策略分析"（AI驱动的板块和龙虎榜策略）
  - "投资管理"（持仓跟踪与实时监测）
- 隐藏方式应该简单、灵活，便于后期根据需要重新启用

### 1.2 技术背景
- 前端框架：Streamlit
- 导航实现：侧边栏按钮 + session_state控制
- 代码结构：主要导航逻辑位于 `frontend/app.py` 文件中

## 2. 设计方案

### 2.1 方案概述
采用配置文件控制方式，通过创建一个功能开关配置文件，来控制各个功能模块的显示与隐藏。这种方式具有以下优点：
- 不需要修改核心代码
- 配置灵活，可随时调整
- 对现有功能无侵入性
- 易于理解和维护

### 2.2 技术实现

#### 2.2.1 创建配置文件
1. 在 `config` 目录下创建 `feature_flags.py` 文件
2. 定义功能开关配置

#### 2.2.2 修改导航逻辑
1. 在 `frontend/app.py` 中导入配置
2. 根据配置控制导航按钮的显示

#### 2.2.3 配置管理
1. 提供配置文件说明
2. 说明如何修改配置以显示/隐藏功能

## 3. 详细实现步骤

### 3.1 创建功能开关配置文件

**文件路径：** `config/feature_flags.py`

```python
"""
功能开关配置
用于控制前端功能入口的显示与隐藏
"""

# 功能开关配置
feature_flags = {
    # 选股板块
    "show_selector_section": False,  # 是否显示选股板块
    "show_main_force": False,        # 是否显示主力选股
    "show_low_price_bull": False,    # 是否显示低价擒牛
    "show_small_cap": False,         # 是否显示小市值策略
    "show_profit_growth": False,     # 是否显示净利增长
    
    # 策略分析
    "show_strategy_section": False,  # 是否显示策略分析板块
    "show_sector_strategy": False,   # 是否显示智策板块
    "show_longhubang": False,        # 是否显示智瞰龙虎
    
    # 投资管理
    "show_investment_section": False,  # 是否显示投资管理板块
    "show_portfolio": False,           # 是否显示持仓分析
    "show_smart_monitor": False,       # 是否显示AI盯盘
    "show_monitor": False,             # 是否显示实时监测
}

# 辅助函数
def is_feature_enabled(feature_name):
    """
    检查功能是否启用
    
    Args:
        feature_name: 功能名称
        
    Returns:
        bool: 功能是否启用
    """
    return feature_flags.get(feature_name, True)
```

## 4. 使用说明

### 4.1 功能隐藏
按照上述实现步骤完成配置后，相关功能的前端入口将被隐藏。

### 4.2 功能启用
如果需要重新启用某个功能，只需修改 `config/feature_flags.py` 文件中的对应配置项为 `True` 即可。

例如，要启用选股板块，只需将：
```python
"show_selector_section": False,
```
修改为：
```python
"show_selector_section": True,
```

### 4.3 部分启用
也可以只启用板块中的部分功能，例如只启用主力选股：
```python
"show_selector_section": True,  # 显示选股板块
"show_main_force": True,        # 显示主力选股
"show_low_price_bull": False,    # 隐藏低价擒牛
"show_small_cap": False,         # 隐藏小市值策略
"show_profit_growth": False,     # 隐藏净利增长
```

## 5. 注意事项

1. **配置生效**：修改配置后，需要重启应用才能生效。
2. **功能完整性**：隐藏前端入口不会影响后端功能的正常运行，只是在前端不显示入口。
3. **安全性**：这种方式只是隐藏前端入口，不是真正的权限控制。如果用户知道直接访问的方式，仍然可以进入功能页面。
4. **向后兼容**：此方案对现有功能无侵入性，可随时回滚。

## 6. 总结

本设计方案通过配置文件控制的方式，实现了前端功能入口的隐藏功能。该方案具有以下特点：

- **简单易用**：只需修改配置文件即可控制功能的显示与隐藏
- **灵活可控**：可根据需要单独控制每个功能模块
- **无侵入性**：不需要修改核心代码，对现有功能无影响
- **易于维护**：配置文件结构清晰，便于理解和维护

通过此方案，可以有效地隐藏指定的前端功能入口，同时保持系统的灵活性和可扩展性。
