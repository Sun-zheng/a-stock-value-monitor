# 飞书多维表格设计

固定 Base 名称：`A股主板低估股票每日分析记录`

固定 Base 配置：`data/feishu_table.json`

程序启动后先读取固定 `base_token` 和 `table_id`，校验字段后按“执行日期 + 股票代码”搜索记录。同一只股票在同一天存在记录时使用 `record_id` 更新；不存在才新增。正式推荐与观察股票均逐股写入，不把多只股票合并在一行。程序不会按天创建 Base。

字段清单由 `src/lark_bitable_client.py` 中的 `LARK_FIELDS` 维护，覆盖执行状态、股票信息、境内全市场基准范围、九维评分、芒格反向失败清单、市场错配、已计价预期、三种独立估值、财务质量、现金流、资产负债、十年持有结论、建议、风险、完整报告、来源、飞书和邮件状态。

所有字段当前使用文本类型。每次交付会为 `LARK_FIELDS` 中的每个单元格写入非空文本；缺失数据明确写“无”或“数据不足”，不得留空或因此加分。邮件发送成功后会再次幂等更新对应记录的邮件状态。后续需要图表时，可在飞书中增加数值派生字段，但不要删除 `执行日期` 或 `股票代码`。
